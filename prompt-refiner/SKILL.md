# Prompt Refiner

> 把模糊的 prompt 变成 AI 精准执行的指令

## 🎯 解决什么问题

写完 prompt，AI 答非所问？输出太烂？反复修改还是不满意？

问题不是"AI 太笨"，而是 prompt 缺少**结构化设计**。大多数人的 prompt 像这样：

> "帮我写一段文案" → AI 不知道什么文案、给谁看、什么风格
> "分析一下这个问题" → AI 不知道用什么框架、从哪个角度

**Prompt Refiner** 用六维诊断法快速定位问题，生成结构化的改进版本。

---

## 🔬 六维诊断法

收到用户 prompt，按以下六个维度打分（1-5 分）：

| 维度 | 问题 | 修复方法 |
|------|------|----------|
| **角色** | 没有定义 AI 身份 | 加 "你是一个 xxx" |
| **任务** | 动词模糊（"分析"vs"总结"） | 用具体动词替代 |
| **约束** | 没有限制条件 | 加格式/长度/风格限制 |
| **示例** | 没有示范 | 提供 1-2 个输入输出例子 |
| **格式** | 没有指定输出结构 | 明确 JSON/Markdown/列表等 |
| **上下文** | 缺少背景信息 | 补充相关背景 |

---

## 📖 使用方法

### Step 1：诊断

读取用户的原始 prompt，逐维打分，找出得分最低的 2-3 个维度。

### Step 2：生成改进版

按以下模板重写：

```markdown
【改进版 prompt】
[角色定义]：你是一个...
[任务]：请完成...
[约束]：必须遵守...
[示例]：例如输入...时输出...
[格式]：以...格式输出
[上下文]：背景是...

完整 prompt：
"""
[最终版本的完整 prompt]
"""
```

### Step 3：解释改动

用表格说明每个维度的原始问题 → 改进方案。

### Step 4：测试建议

提供 2 个测试用例，验证改进是否有效。

---

## ⚡ 快速模式（30 秒版）

如果用户只需要快速修复，不需要完整报告，用这个简化版：

```
原始 prompt：[用户的 prompt]
❌ 主要问题：...
✅ 改进后：
"""
[3 句话以内的改进版]
"""
改进原因：...
```

## 示例

**用户输入：**
> "帮我分析一下竞争对手"

**诊断结果：**
| 维度 | 得分 | 问题 |
|------|------|------|
| 角色 | 1 | 没有定义分析师身份 |
| 任务 | 3 | "分析"太模糊 |
| 约束 | 1 | 没有范围/深度限制 |
| 示例 | 1 | 无参考案例 |
| 格式 | 1 | 没有输出格式 |
| 上下文 | 1 | 不知道是什么行业/产品 |

**改进版：**
```
【改进版 prompt】
你是一个高级竞品分析师，专注于[行业/领域]。

请分析以下竞争对手：
[竞争对手名称/产品]

分析维度（每项 200 字）：
1. 产品定位
2. 核心功能
3. 定价策略
4. 用户评价

输出格式：Markdown 表格 + 关键结论（3 条）

背景：[相关市场背景]

完整 prompt：
"""
你是一个高级竞品分析师，专注于电商服装行业。请分析 [竞争对手名称] 的：
1. 产品定位（50字）
2. 核心差异化功能（列出 Top 3）
3. 定价策略（区间 + 促销模式）
4. 用户评价总结（正面/负面各 3 条）

输出格式：
## [竞争对手名]
### 产品定位
...
### 核心功能
...
[Markdown 表格]
### 关键结论
1. ...
2. ...
3. ...
"""
```

---

## ⚠️ 注意事项

- 如果用户 prompt 已经很好（6 维都 ≥ 4），直接说"这个 prompt 已经很完善，无需改进"
- 不要过度设计——简单任务用简单 prompt，不要套用复杂框架
- 角色定义要具体但不能矛盾（如"专业且幽默"要谨慎）
- 示例（Few-shot）是最强修复手段，优先考虑加示例

## 🔍 验证步骤

用以下测试 prompt 验证 skill 是否正常工作：

```
输入："帮我写一封道歉邮件"
期望：输出包含诊断 + 改进版 + 解释的完整报告，或快速模式的简洁版本
```
